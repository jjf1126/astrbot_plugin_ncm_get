# AstrBot 插件：ncm_get (网易云音乐解析增强)

## 插件简介
`astrbot_plugin_ncm_get` 是一款专为 AstrBot 开发的音乐链接解析与上下文增强插件。当用户在聊天中发送网易云音乐的链接时，本插件会自动访问该网页，提取歌曲的核心信息（包括歌曲名称、歌手/作者、完整歌词等），并将这些提取到的信息无缝注入到当前的消息上下文中。
通过这种方式，大语言模型（LLM）能够“看懂”用户分享的音乐，从而围绕该歌曲的歌词、创作背景或歌手信息与用户进行更深入、准确的对话。同时，插件支持配置网易云音乐的 Cookie，以获取更完整的页面数据。

## 功能说明
*   **链接自动识别与解析**：支持识别消息中的网易云音乐链接，提取网页 DOM 元素中的关键文本（歌名、歌手、歌词）。
*   **上下文自动注入**：将抓取到的歌曲信息自动追加到用户的原始消息请求中，辅助 LLM 生成回复，并同步保存至历史会话上下文中。
*   **Cookie 身份验证支持**：支持设置和更新网易云音乐的 Cookie，以应对部分需要登录才能查看完整信息的网页。
*   **手动/自动双模式**：既可以通过专门的指令手动触发解析，也可以在普通对话中自动拦截并解析链接。

## 插件流程
本插件的核心工作流程依赖于 AstrBot 的事件钩子与异步网络请求库，具体内部逻辑如下：

1.  **事件监听与拦截**：
    *   插件通过 `@filter.on_llm_request()` 钩子监听即将发送给大语言模型（LLM）的请求。
    *   当检测到用户的输入文本中包含网易云音乐的 URL（如 `music.163.com`），或用户显式调用了 `/ncm_get` 指令时，触发解析流程。
2.  **网页请求与 Cookie 附加**：
    *   插件内部使用 `aiohttp` 异步发起对目标音乐链接的 HTTP GET 请求。
    *   如果用户事先通过 `/ncm_cookie` 存储了 Cookie，插件会在请求头（Headers）中携带该 Cookie，以模拟登录状态获取完整的网页文档。
3.  **DOM 元素解析**：
    *   获取到 HTML 响应后，插件利用 `beautifulsoup4` 库对网页 DOM 树进行解析。
    *   精准定位并提取歌曲名称（Title）、歌手（Artist）以及隐藏或展开的完整歌词（Lyrics）等纯文本信息。
4.  **上下文注入与 LLM 传递**：
    *   解析完成后，插件会将提取到的结构化信息（例如：“[系统附加信息] 用户分享了歌曲《XXX》，歌手：XXX，歌词内容：...”）动态追加到 `ProviderRequest` 的请求文本（prompt）或系统提示词中。
    *   经过增强的请求继续传递给 LLM，LLM 结合这些信息生成回复。
5.  **历史记录保存**：
    *   AstrBot 的核心会话管理器会自动将这段包含歌曲信息的完整上下文保存到当前用户的对话历史中，确保后续的多轮对话仍具备该歌曲的记忆。

## 使用方法

### 1. 自动解析模式
在日常聊天中，直接向机器人发送包含网易云音乐链接的消息即可。
*   **用户**：“听听这首歌，你觉得歌词写得怎么样？ https://music.163.com/#/song?id=xxxxxx”
*   **机器人**：（插件在后台提取歌词并注入上下文）“这首《XXX》的歌词非常打动人，尤其是那句‘...’，表达了...”

### 2. 手动解析指令 (`/ncm_get`)
如果你只需要单纯地让机器人获取某首歌的信息并作为背景知识，可以使用手动指令：
*   **语法**：`/ncm_get <音乐链接>`
*   **示例**：`/ncm_get https://music.163.com/#/song?id=123456`
*   **说明**：机器人会立即解析该链接，并回复解析成功的基础信息，同时将详细歌词存入上下文。

### 3. 设置 Cookie (`/ncm_cookie`)
为了获取某些特殊歌曲的完整信息，你可以设置你的网易云音乐 Cookie。
*   **语法**：`/ncm_cookie <你的Cookie字符串>`
*   **示例**：`/ncm_cookie MUSIC_U=xxxxx; __csrf=yyyyy;`
*   **说明**：设置成功后，插件会在后续的解析请求中自动携带此 Cookie。更新 Cookie 也是使用同一条指令。
